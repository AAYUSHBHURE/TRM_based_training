An educational AI gaming platform powered by the **Tiny Recursive Model (TRM)** — a parameter-efficient transformer architecture that achieves deep reasoning by reusing a single block T times instead of stacking unique layers.

> **Core idea:** instead of building a bigger network, reuse the same small network T times.
> More cycles = better accuracy. Zero extra parameters.

---

## What is TRM?

Standard transformers scale depth by stacking N unique layers (N × parameters).
TRM scales depth by recycling one shared block T times (constant parameters):

```
Standard Transformer:
  Input → Layer₁ → Layer₂ → … → LayerN → Output
  Parameters: N × |Layer|  (grows with depth)

TRM:
  Input → [Block] → [Block] → [Block] → Output
            t=1       t=2       t=3
  Parameters: |Block|  (constant, regardless of T)
```

**Key result (Maze task):**
T=2 → 64.5% accuracy | T=4 → ~80% | T=6 → ~87%  — same 2M parameter model.

Research paper: [arXiv 2510.04871v1](2510.04871v1.pdf) — included in this repo.

## Quick Start — Run the Demo

### Step 1 — Prerequisites

- Python 3.8 or higher
- pip package manager
- Git

Check your Python version:
```bash
python --version
```

### Step 2 — Clone the Repository

```bash
git clone https://github.com/AAYUSHBHURE/TRM_based_training.git
cd TRM_based_training
```

### Step 3 — Create a Virtual Environment (Recommended)

**Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**macOS / Linux:**
```bash
python -m venv .venv
source .venv/bin/activate
```

### Step 4 — Install Dependencies

```bash
pip install -r requirements.txt
```

This installs: `streamlit`, `torch`, `numpy`, `plotly`, and all other dependencies.

For GPU support (optional, speeds up model inference):
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

### Step 5 — Launch the Demo

```bash
streamlit run app.py
```

Then open your browser at: **http://localhost:8501**

## Architecture Overview

### TinyNet (The Shared Block)

```
Input [B, L, D]
  └─ RMSNorm → Multi-Head Self-Attention (RoPE) → Residual Add
  └─ RMSNorm → SwiGLU FFN → Residual Add
Output [B, L, D]
```

- **RMSNorm**: faster than LayerNorm, used in LLaMA/Mistral
- **RoPE**: rotary positional encoding — better length generalization
- **SwiGLU**: `W2(SiLU(W1·x) * W3·x)` — gated activation, 10-20% better than ReLU

### Latent Recursion (The Core Loop)

```python
# n_latent inner z-updates (reasoning mode, conditioned on input x)
for _ in range(n_latent):
    z = TinyNet( Linear([x, y, z]) )

# 1 outer y-update (solution refinement mode, no x)
y = TinyNet( Linear([y, z]) )
```

### Deep Recursion (Training with Gradient Control)

```python
# T-1 warmup cycles — no gradients (avoids BPTT memory explosion)
with torch.no_grad():
    for _ in range(T - 1):
        y, z = latent_recursion(x, y, z)

# Final cycle — gradients enabled for backprop
y, z = latent_recursion(x, y, z)
logits = output_head(y)
```

| T (cycles) | Accuracy | Parameters |
|------------|----------|------------|
| T=2 | 64.5% | 2M |
| T=4 | ~75–80% (projected) | 2M |
| T=6 | ~82–87% (projected) | 2M |

This is the core TRM novelty: increase T at inference time for free accuracy gains.
